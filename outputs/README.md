# Pipeline Outputs

This directory holds the four artifacts produced by `src/run_pipeline.py`. All files are gitignored because they are regenerable from the source data. This README documents each so a reader of the repo can understand what the pipeline produces without running it.

## Files

| Filename | Rows | Granularity |
|---|---:|---|
| `submission_final.csv` | 12,000 | One row per test claim |
| `cv_results.csv` | 14 | One row per evaluated model (Step 5) |
| `optimization_results.csv` | 3 | One row per top model (Step 6) |
| `feature_importance.csv` | ~68 | One row per encoded feature |

## `submission_final.csv` — final predictions

| Column | Type | Description |
|---|---|---|
| `claim_number` | int | Test row identifier, preserved exactly from the input |
| `subrogation` | int | Binary prediction: 1 = predicted subrogation opportunity, 0 = no |

Predictions are generated by applying the best CV model (Tree Super Stack on the validated run) refit on the full training data, with predictions thresholded at the tuned decision threshold (~0.44 on the validated run).

The pipeline asserts that the output row count matches the test input row count — if these don't match, the script raises rather than producing a misaligned submission.

## `cv_results.csv` — Step 5 model comparison

One row per evaluated model. Columns:

| Column | Description |
|---|---|
| `F1` | Mean F1 across 5 folds, evaluated at each fold's F1-optimal threshold |
| `Best_Threshold` | Mean best-F1 threshold across folds |
| `AUC` | Mean ROC-AUC across folds |

Sorted by F1 descending. Used to identify the top 3 models for Step 6 optimization.

## `optimization_results.csv` — Step 6 SMOTE optimization

One row per model from `top_models_for_optimization` (default: Tree_Super_Stack, XGBoost, Stacked_Hetero_XGBmeta).

| Column | Description |
|---|---|
| `Mean_F1` | Mean F1 across 5 folds with SMOTE inside the training fold |
| `Std_F1` | Standard deviation of fold-level F1 |
| `Mean_Threshold` | Mean best-F1 threshold across folds (used for final test predictions) |
| `Precision` | Mean precision at the F1-optimal threshold |
| `Recall` | Mean recall at the F1-optimal threshold |
| `ROC_AUC` | Mean ROC-AUC |
| `PR_AUC` | Mean precision-recall AUC |

The headline metric the README reports is `Mean_F1` of the winning model from this table. On the validated run that's `Tree_Super_Stack` with Mean_F1 = 0.5971, std = 0.0139.

## `feature_importance.csv` — Step 4 LightGBM wrapper importance

One row per encoded feature (categorical features expand into multiple one-hot columns).

| Column | Description |
|---|---|
| Feature name (index) | Encoded feature name. `remainder__*` = numeric passthrough; `cat__<col>_<value>` = one-hot encoded category |
| `importance` | LightGBM split-gain importance |

Sorted descending. The top 60 features are selected for downstream modeling; the rest are dropped.

## Validated headline metrics

On a clean local end-to-end run:

| Metric | Value |
|---|---|
| Headline CV Mean_F1 (Tree Super Stack) | **0.5971** |
| Std F1 across folds | 0.0139 |
| CV ROC-AUC | 0.8383 |
| Tuned threshold | 0.442 |
| Test predictions | 12,000 |
| Test positive rate | ~31% |

The `submission_final.csv` produced on this run flags 3,737 of 12,000 test claims as predicted subrogation opportunities.

## A note on the in-sample refit

The pipeline also prints in-sample refit metrics (F1 ~0.67, AUC ~0.90) after fitting the best CV model on the full training data. These are **not the headline metrics** — they are computed on the same data the model was fit on and are optimistically biased. They exist as a sanity check and to produce the final test-prediction model. The README and methodology.md report only the CV F1 as the honest result.

## Regenerating the outputs

From the repository root:

```bash
python src/run_pipeline.py
```

The pipeline writes all four files to this directory and reports the headline CV F1 in the final log line.
